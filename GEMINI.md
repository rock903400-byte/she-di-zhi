# 社地址地圖顯示器 (Excel to Map Viewer)

這是一個將 Excel 地址資料轉換為互動式地圖標註的工具。

## 專案結構
- `社地址.xlsx`: 原始資料來源。
    - B 欄: 社名
    - N 欄: 地址 (支援帶有郵遞區號與備註的格式)
- `map_viewer.html`: 網頁主程式，包含資料讀取、地理編碼與地圖呈現。

## 技術棧
- **Leaflet.js**: 地圖呈現與標記管理。
- **SheetJS (xlsx.js)**: 瀏覽器端 Excel 檔案讀取。
- **Nominatim API (OpenStreetMap)**: 免費地理編碼服務。

## 關鍵功能與邏輯

### 1. 地址清理 (Address Cleaning)
- 自動移除地址開頭的 3-6 位數郵遞區號。
- 將全形數字轉為半形以提升搜尋辨識度。
- 移除括號內的備註字串。

### 2. 多層級降級搜尋策略 (Geocoding Fallback)
針對 Nominatim 對台灣地址支援度不一的問題，實施以下搜尋順序：
1. **完整地址**: `[路名門牌], [鄉鎮區], [縣市], 台灣`
2. **路名搜尋**: `[路名], [鄉鎮區], [縣市], 台灣` (移除號碼)
3. **路名去村里**: 移除路名前綴的村里名後搜尋。
4. **極簡路名**: `[路名], [縣市], 台灣`
5. **村里搜尋**: `[村里], [鄉鎮區], [縣市], 台灣` (針對偏鄉無路名地址)
6. **行政區中心**: `[鄉鎮區], [縣市], 台灣` (最終保底)

### 3. API 頻率限制 (Rate Limiting)
- Nominatim 要求每秒最多 1 次請求。
- 程式在 `fetchCoords` 函數中強制執行 **1.2 秒** 的全域延遲，確保多重備案搜尋時不會觸發 IP 封鎖。

## 使用說明
1. 開啟 `map_viewer.html`。
2. 點擊「選取檔案」並選擇 `@社地址.xlsx`。
3. 等待程式逐一處理（每筆約 1.2~5 秒不等，視備案嘗試次數而定）。
4. 處理完畢後，點擊地圖標記可查看詳細資訊。
